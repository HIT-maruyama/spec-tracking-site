# 仕様駆動開発管理サイト

仕様駆動開発（Spec-Driven Development）の実践記録を管理・表示する静的Webアプリケーションです。

## 概要

このサイトは、仕様駆動開発プロジェクトの履歴、CI結果、レビュー指摘、開発効果メトリクスを構造化された形式で記録し、将来の標準化取り組みを支援します。

## 主要機能

- **プロジェクト管理**: メタデータ、CI結果、効果メトリクスを含む仕様駆動開発プロジェクトの追跡
- **レビュー追跡**: 構造化された分類システムによるコードレビュー指摘の記録・分類
- **CI統合**: lint、契約テスト、カバレッジ、SBOMステータスを含む継続的インテグレーション結果の監視
- **効果測定**: 効率性、品質、人間の手間、引継ぎ正確性における開発効果の定量化
- **データポータビリティ**: データバックアップのためのエクスポート機能

## 技術スタック

- **HTML5**: セマンティックマークアップ
- **CSS3**: レスポンシブデザイン
- **Vanilla JavaScript (ES6+)**: フレームワークなし、モダンブラウザAPIの活用
- **LocalStorage API**: クライアントサイドデータ永続化
- **JSON**: ストレージとエクスポートのためのデータフォーマット

## ブラウザ要件

- **最小要件**: ES6サポート、localStorage、Fetch API
- **対象ブラウザ**: Chrome 60+、Firefox 60+、Safari 12+、Edge 79+

## 使用方法

### 基本的な使い方

1. **プロジェクト管理**
   - メインページでプロジェクト一覧を確認
   - 「プロジェクト追加」ボタンで新しいプロジェクトを作成
   - プロジェクト名をクリックして詳細ページに移動

2. **CI結果の記録**
   - プロジェクト詳細ページの「CI結果」タブを選択
   - 「CI結果追加」ボタンでCI実行結果を記録
   - lint結果、テスト結果、カバレッジ等を記録可能

3. **レビュー指摘の管理**
   - プロジェクト詳細ページの「レビュー指摘」タブを選択
   - 「レビュー指摘追加」ボタンで新しい指摘を記録
   - フィルタリング・ソート機能で指摘を整理

4. **効果メトリクスの記録**
   - プロジェクト詳細ページの「効果メトリクス」タブを選択
   - 効率性、品質、人間の手間、引継ぎ正確性の各指標を記録

### データ管理

- **データ永続化**: ブラウザのlocalStorageに自動保存
- **データエクスポート**: メインページの「データエクスポート」ボタンでJSONファイルをダウンロード
- **初期データ**: 初回アクセス時にサンプルデータを自動読み込み

## ファイル構成

```
/
├── index.html              # プロジェクト一覧ページ
├── project-detail.html     # プロジェクト詳細ページ
├── styles.css              # グローバルスタイル
├── app.js                  # メインアプリケーションロジック
└── data/
    ├── projects.json       # サンプルプロジェクトデータ
    └── review_log.json     # サンプルレビュー指摘
```

## 分類システム

レビュー指摘は以下の分類体系で管理されます：

- **プロセス**: planning, design, implementation, testing, documentation
- **ドキュメント種別**: spec, design_doc, source_code, test_case等
- **カテゴリ**: requirements_gap, implementation_bug, security_issue等
- **重要度**: high, medium, low
- **ステータス**: Open, InProgress, Resolved, Verified, Deferred, Rejected

## 開発・デプロイメント

### ローカル開発

```bash
# 静的サーバーを起動
python -m http.server 8000
# または
npx serve .
```

### GitHub Pagesデプロイ

1. GitHubリポジトリにプッシュ
2. リポジトリ設定でGitHub Pagesを有効化
3. ソースを「Deploy from a branch」に設定
4. ブランチを「main」、フォルダを「/ (root)」に設定
